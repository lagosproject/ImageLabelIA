# Security Policy

## Supported Versions

The security team actively monitors and provides security updates for the following versions of **Image Label IA**:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

We take the security of **Image Label IA** seriously. If you believe you have found a security vulnerability, please **DO NOT** open a public issue. Doing so exposes users to potential exploits before a patch is available.

Instead, please report vulnerabilities responsibly by following these steps:

1. **Email the Maintainers**: Send a detailed email to `security@lakescorp.com` (or the maintainer's primary contact).
2. **Details to Include**:
   - A description of the vulnerability and its potential impact.
   - Step-by-step instructions (or a proof-of-concept script/exploit) to reproduce the issue.
   - Details of the system environment you used (OS version, hardware, stack versions).
3. **Response Timeline**:
   - You will receive an acknowledgment of your report within 48 hours.
   - We will coordinate a disclosure timeline and keep you updated as we work on a fix.
4. **Credit**:
   - Once the issue is resolved and a release is published, we will gladly credit you for your responsible disclosure (if desired).

Thank you for helping keep this project secure!
